# Checklist: UI Review

## Purpose

Use this checklist to review whether a screen is aligned with IDDS-inspired government UI standards.

---

## 1. Page Purpose

- [ ] The page purpose is immediately clear.
- [ ] The H1 accurately describes the page.
- [ ] The description helps users understand what they can do.
- [ ] The primary action is obvious.
- [ ] Secondary actions do not compete with the primary action.

---

## 2. Layout and Hierarchy

- [ ] Information is ordered by priority.
- [ ] Related items are grouped.
- [ ] Layout is not overly dense.
- [ ] Important information is visible without unnecessary scrolling.
- [ ] The page uses consistent spacing.
- [ ] The layout works on mobile, tablet, and desktop.

---

## 3. Typography

- [ ] One clear H1 exists.
- [ ] Heading levels are logical.
- [ ] Body text is readable.
- [ ] Font sizes follow the typography scale.
- [ ] Long paragraphs are avoided.
- [ ] Metadata and helper text are readable.

---

## 4. Color

- [ ] Color is used semantically.
- [ ] Brand color is not overused.
- [ ] Status colors are consistent.
- [ ] Text has sufficient contrast.
- [ ] The page does not rely on color alone.
- [ ] Gradients are avoided for typography.

---

## 5. Components

- [ ] Standard components are used where possible.
- [ ] Buttons have consistent hierarchy.
- [ ] Form controls have visible labels.
- [ ] Status badges include text.
- [ ] Modals are used only for focused decisions.
- [ ] Tables are used for structured data.
- [ ] Tooltips are not used for critical information.

---

## 6. States

- [ ] Loading state exists.
- [ ] Empty state exists.
- [ ] Error state exists.
- [ ] Success state exists where relevant.
- [ ] Disabled state exists where relevant.
- [ ] No-result state exists for filtered/search pages.
- [ ] Destructive action confirmation exists.

---

## 7. Copywriting

- [ ] UI copy uses Indonesian when appropriate.
- [ ] Button labels use clear verbs.
- [ ] Error messages are specific.
- [ ] Success messages explain what happened.
- [ ] Empty states explain next action.
- [ ] Technical jargon is avoided for citizen-facing screens.

---

## 8. Anti AI Slop

- [ ] No random gradients.
- [ ] No excessive decorative icons.
- [ ] No fake dashboard metrics without context.
- [ ] No inconsistent cards.
- [ ] No low-contrast gray text.
- [ ] No generic SaaS hero section inside admin panel.
- [ ] No meaningless visual effects.
